# 根据这个 NanoFab Knowledge Book 里的内容，做一个用来学习微纳制造的网站。

这个网站的基础就是这本书一章一章的内容。网站的学习过程，当然也是一章一章地进行。这个学习网站的内容其实有两个版本：一个是静态的、针对所有人的标准版；另一个是动态的、针对每个学习者的个人定制版。

## 如何形成个人定制版呢？定制的基础主要有两个：

1. 用户的学习动机
   他想要通过学习来完成什么目标。

2. 用户本身的基础
   他对 Nano Fabrication 已经有了什么样的了解。比如他是材料出身、光学出身，还是计算机出身？

这决定了他在学习时，哪些内容是显而易见的，不需要花太多时间；而哪些内容是他需要更多细节的。

因此，这些学习内容应该以定制化的方式呈现在他面前，不再是原来一成不变的设计，而是根据他个人的画像，生成一套定制化的学习脉络。首先是确定这样一个脉络，然后在脉络的每一个环节，提供针对他个人基础和学习目的而定制的内容。

AI 智能化的另一个方面，是解决学生在学习过程中无法预被测的实时需求。

学生在看某一句话、某一个段落或者某幅图时，难免会产生疑惑，想要深入了解更多内容。在传统的模式下（这里指一年前的互联网时代），学生需要打开 Google 搜索，或者翻找其他教科书，甚至是切换到 AI 对话框去提问。

而在我们的学习内容中，当学生高亮一段令他困惑的文字并表达深入了解的意图时，系统会直接根据他的需求和背景，为他生成定制化的解释。

当然，这种定制化不能太过跳脱。因为人们在理解新知识时，首先需要回顾知识基础，把逻辑链打通。所以我们在做个性化生成时，一定要注意不能用力过猛、矫枉过正。

## 这种个人定制并不是一成不变的。

它包含两部分：
1. 初始基础：用户刚开始学习时的动机和基础，这是个人定制的一部分。
2. 动态行为：用户在学习过程中的行为是定制化的另一部分。

也就是说，系统会关注：
- 用户对哪些内容付出了更多注意力，对哪些内容有更多疑问。
- 用户在哪里停留的时间更长，哪里让他走神，或者什么样的内容能吸引他的注意力。
- 在做 quiz 时，哪些题做对了，哪些题做错了，以及他是否会回顾学过的东西。

总而言之，用户在页面上产生的所有动作、行为模式和数据，都会形成新的数据，成为个人画像的一部分。而这部分正是 AI 时代个人定制学习最重要的变革和价值所在。

## 所以在我们设计这个学习系统的时候，有几个部分是我们需要做的工作。

### 首先是第一部分：了解学习的目的和学习者的背景。

1. 学习目的
   可以让学习者自己表述，或者提供一些标签供其选择。

2. 学习者背景
   (a) 素材收集：最好能让学习者上传简历和成绩单，并描述一下自己。此外，还可以提供他的证书、论文，或者个人 profile 网站（如 Google Scholar、ORCID 等）。总之，信息越全越好。
   (b) 画像生成：根据这些目的和背景生成用户画像。
   (c) 主动了解：我们甚至可以主动去了解用户的背景信息，比如他所在实验室目前关注的研究方向，这也是个人背景的一部分。
   
### 系统的另外一部分，是根据已有的定制背景来呈现要学习的内容。

我们已经有了一个静态的版本。那么在呈现动态版本的时候，可以在静态版本的基础上有所增删。为了实现信息的快速获取，删除内容肯定比增加内容要快。因此，在我们生成静态版本时，应该遵循两个原则：

1. 内容越多越好，越丰富越好（在保持准确性的前提下）：
   这样就有现成的内容供调用，尽可能减少或缩短实时搜索和归纳的过程。

2. 静态版本不应只有 Markdown 或 HTML 格式：
   它应该有一个非常结构化的状态。总之，要让动态版本能够尽可能高效地获取静态知识库里的内容。


### 系统的另一个部分是对学习者在学习过程中行为数据的收集。

这些行为数据该如何收集，我现在还没有确定的方案。你可以跟 AI 讨论一下，看看有哪些数据是：
1. 有价值且值得收集的
2. 技术上可以实现的

## 具体的大致操作步骤：

1. 定义学习动机与背景
   以你自己为例，先定义一下学习的动机和背景。比如动机方面，可以先以“X射线衍射光学元件”为例；学习背景则结合你自己的知识储备，把能提供的信息都提供一下，生成一个用户画像的例子。

2. 系统载体与开发环境
   在制作这个系统时，系统的载体可以暂时考虑采用 HTML 状态，因为最终是要呈现在网站上的，所以可以先用 HTML 来开发。
   关于开发工具，比如 Claude Code 或者 Open Code，应该可以在窗口内直接预览（Preview）你设计的 HTML 系统。

3. 内容架构与智能学习设计
   呈现内容时，我们可以从最基础、最核心的部分开始。比如静态数据源分了十几章，那么我们的智能学习系统也可以对应分为十几个章节。每个章节包含用户定制化的学习内容。

4. 增强学习效率的工具
   为了让 Nano Fabrication 的学习者能够更高效、更有效地学习，在每一部分内容之后，可以加入以下设计：
   (a) 要点卡片：提炼核心知识点。
   (b) 小测试（Quiz）：涵盖几个重要方面，通过层层递进的问题引导用户深入思考。
   (c) 思考练习：通过这些小测试帮助用户回顾并巩固前面学到的内容。
   
## Tips

这个系统的设计过程一定会涉及非常多的迭代和反复修改，所以它的上下文一定会很长，你一定要做好 continuity file 的维护。有问题的时候一定要问我，不要一条路走到黑。

那这个页面当然也需要有一个用户注册系统，因为我们是要给很多不同用户使用的。有了用户注册，才能保存用户的画像（用户 profile）
